# 교사용 데스크톱 위젯 모음 — 설계 문서

- 작성일: 2026-06-29
- 대상: 선일초등학교 교사용 윈도우 바탕화면 위젯 (동료 교사 배포 목표)
- 레퍼런스: `reference/TeacherNativeWidgets` (PySide6 + PyInstaller 기존 앱)

---

## 1. 목표와 제약

- **목표:** 교사가 수업·학급 운영에 바로 쓰는 바탕화면 위젯 모음을 만들어 **동료 교사에게 배포**한다.
- **배포 전제:** 파이썬 미설치 PC에서도 `.exe` 더블클릭만으로 실행. 설치·안정성·설정 편의가 중요.
- **개인정보 원칙:** 출결·상담기록 등 학생 관련 민감 데이터는 **교사 PC 로컬에만** 저장하고 외부로 절대 전송하지 않는다.

## 2. 기술 스택

- **데스크톱:** Python 3.x + **PySide6 (Qt Widgets)** → **PyInstaller** 단일 `.exe` 배포 (레퍼런스와 동일, 검증된 경로).
- **서버:** 기존 **Vercel 웹앱에 읽기 전용 프록시 API**를 추가해 외부 데이터·API 키를 서버에 은닉.
- **선정 이유:** 레퍼런스를 학습·응용 자산으로 그대로 활용 가능, 프레임리스·반투명·트레이·네트워크 등 교사 위젯에 필요한 기능이 Qt 표준 제공, exe 단일 배포가 검증됨, 학습 곡선이 완만.

## 3. 핵심 아키텍처 원칙 — 두 개의 데이터 세계

| | 외부형 | 로컬형 |
|---|---|---|
| 해당 위젯 | 시간표, 주간학습계획, 급식, 날씨 | 출결, 상담기록, 체크, 할일 등 |
| 데이터 출처 | Vercel 프록시 → 외부 공개 API | 교사 PC 로컬 |
| 성격 | 공개·읽기전용·캐시 | 민감·개인정보 |
| 외부 전송 | O (공개 데이터만) | **절대 금지** |

이 분리가 **개인정보 안전 + 배포 안전**의 토대. 모든 외부 호출은 `data_remote` 한 곳을 거치고, 모든 로컬 민감 데이터는 `data_local`을 거친다.

## 4. 내부 모듈 구조

```
TeacherWidgets/
├─ core/
│  ├─ base_widget.py     # 프레임리스·반투명·드래그이동·리사이즈·멀티모니터·숨기기·우클릭메뉴
│  ├─ responsive.py      # 크기→폰트/여백 반응형 + 내용 구간(breakpoint) 시스템
│  ├─ config_store.py    # config.json 단일 상태 저장소 읽기/쓰기
│  ├─ data_local.py      # 로컬 민감 데이터 (외부 전송 금지)
│  ├─ data_remote.py     # Vercel 프록시 호출 + 오프라인 캐시 + 에러처리
│  └─ launcher.py        # 시스템 트레이: 위젯 on/off·설정·종료
├─ widgets/              # 각 위젯 = core 상속 독립 모듈
├─ assets/              # 아이콘·예비종 사운드·폰트
└─ main.py              # 진입점: 런처 기동 + 저장된 위젯 복원
```

설계 원칙:
- 각 위젯은 "무엇을 그리고 어떤 데이터를 쓰는가"만 담당. 이동·리사이즈·저장·반응형은 `base_widget`이 전담 → 위젯 코드가 짧고 독립적으로 테스트 가능.
- `config.json` 단일 저장소(레퍼런스 계승). 단, 출결/상담 등 민감·대용량은 별도 로컬 파일로 분리 가능하게 설계.

## 5. 전 위젯 공통 기능 (base_widget)

1. **크기 조절 + 반응형:** 크기에 따라 폰트·여백·요소 배치 재계산. 단순 확대가 아님.
2. **내용 구간(breakpoint):** 폭/높이가 기준을 넘으면 표시 정보량 자체가 바뀜 (예: 오늘 → 오늘+내일 → 주간).
3. **자유 이동 + 멀티모니터:** 가상 데스크톱 좌표 기반, 특정 모니터 배치·저장.
4. **숨기기/다시 띄우기:** `visible` 플래그. 닫아도 설정 보존, 트레이/런처에서 재호출.
5. **위치·크기·설정 자동 저장:** `config.json`.

## 6. 위젯 카탈로그 (총 15종, 사진 제외)

### 외부형 (Vercel 프록시 경유, 읽기 전용)
1. **시간표** — 학급/특별실/전담 선택(1회 저장식), 시간표 JSON 표시. 클릭 시 웹앱을 **앱 모드 전용 창**으로 열고 떠 있으면 맨 앞으로. 오프라인 캐시. 갱신: 시작 시 + 주기적.
2. **주간학습계획** — 학교 공유 주간계획(대상 선택 없음). 날짜별 카드, 학사일정/부서 태그, 오늘 강조. **크기 구간**: 오늘 / 오늘+내일 / 주간 전체(+여유 시 교장·교감 말씀). 클릭 시 웹앱 앱모드.
3. **급식** — 나이스 교육정보 개방 포털 급식식단정보 API. 메뉴 + 칼로리 + 알레르기. 학교코드 기본 선일초(변경 가능). 크기 구간으로 오늘/주간.
4. **날씨 + 미세먼지** — 기상청 단기예보 + 에어코리아 API. 오늘 날씨 + 내일 간략 + PM10/PM2.5. 위치 기본 선일초(변경 가능). 오프라인 캐시.

### 로컬형 (PC 로컬, 외부 전송 없음)
5. **출결** — 행=번호(남 1~15 / 여 51~65, 합 30칸; 미달 칸 공란), 열=1~31일(요일·주말 음영) + 수업일수. 셀=나이스 16기호(결석/지각/조퇴/결과 × 질병/미인정/기타/출석인정). 체험학습=출석인정결석(△). 입력 ① 셀 클릭, ② 자연어("7번 6월29일 결석", 날짜 없으면 오늘? 확인 팝업). 월 이동 ◀▶. 월별 Excel 출력(전체 월 / 현재 월). 자주 쓰는 결석·체험학습 빠른 입력, 지각·조퇴는 상세 선택.
6. **상담기록** — 빠른 타이핑 → 각 기록 = **원문(RAW) + 로컬 규칙 정제본 + 타임스탬프(자동·필수)**. 정제는 외부 전송 없는 규칙 기반(타임스탬프 부착, "~~를 지도함" 템플릿화, 말투·띄어쓰기 정리). 시간 역순 열람, Excel 출력. (향후 AI 옵션 구조만 열어둠, 기본 off)
7. **체크** — 공유 학급 구성(남 N·여 M, 남 1~/여 51~)으로 번호 그리드. 번호별 체크박스, 제출 카운트, 초기화. 제목(용도) 편집 가능. **여러 개 동시 사용**.
8. **할일/투두** — 자유 항목 체크리스트.

### 레퍼런스 개선형
9. **타이머** — 디자인·기능 개선.
10. **시계** — 시각 + **날짜·요일** 표시.
11. **메모** — 여러 개 동시 사용.
12. **디데이** — D-day 카운트.
13. **캘린더** — 달력.
14. **QR 생성기** — 텍스트/URL → QR.
15. **교시 타이머(개선)** — config 교시 시각 기반 카운트다운. **교시 시작 1분 전(설정 가능) 예비 알림**: 예비종 소리 + "곧 수업 시작" 큰 화면 표시를 **약 30초 동시 제공, 교사가 중간에 즉시 끄기 가능**. 학생 대상(교실 TV·프로젝터).

## 7. 서버(Vercel) 프록시 설계

- 기존 웹앱에 **읽기 전용 엔드포인트**만 추가. 예: `GET /api/timetable?target=...`, `/api/weekly-plan`, `/api/meal?date=...`, `/api/weather`.
- 모든 외부 API 키(Firebase 접근, 나이스 개방, 기상청, 에어코리아)는 **서버 환경변수**에만 보관 — 배포되는 위젯에 절대 포함하지 않음.
- 서버가 정제된 JSON만 반환 → 위젯과 원본 DB/API 구조 분리. 원본 구조가 바뀌어도 위젯 재배포 불필요.

## 8. 에러 처리 & 오프라인

- 외부형: 호출 실패 시 마지막 캐시 표시 + "오프라인/갱신 실패" 표식. 빈 화면 금지.
- 로컬형: 저장 실패·파일 손상 대비 백업본 유지, 손상 시 복구 시도.
- 전역: 한 위젯의 예외가 다른 위젯·런처를 죽이지 않도록 위젯 단위 격리.

## 9. 배포

- PyInstaller 단일 `.exe` + `_internal` 묶음 (레퍼런스 방식).
- 시작프로그램 등록/해제 `.bat` (레퍼런스 방식 계승).
- `config.json`은 데이터 폴더에 분리 보관(업데이트 시 설정 보존).

## 10. 개발 단계 (로드맵)

- **Phase 0 — 기반:** core 공통 엔진(base_widget, responsive, config_store, launcher) + PyInstaller 배포 골격. 반응형·멀티모니터·exe 배포를 끝까지 검증. (사용자 위젯 0개)
- **Phase 1 — MVP:** 순수 로컬 저위험 3종 **시계 · 타이머 · 메모**. 전체 파이프라인을 작은 위젯으로 증명, 동료 첫 배포.
- **Phase 2 — 양쪽 세계 증명:** 외부형 대표 **시간표** + 로컬형 대표 **체크**.
- **Phase 3 — 외부 연동 마무리:** 주간학습계획 · 급식 · 날씨.
- **Phase 4 — 로컬 데이터(난도 최상):** 출결 · 상담기록 (자연어 입력·Excel·민감정보).
- **Phase 5 — 나머지 개선형:** 디데이 · 캘린더 · QR · 할일 · 교시 타이머(예비종).

**진행 방식:** 각 Phase를 독립 사이클(spec → 구현)로 진행. 지금은 **Phase 0 + Phase 1**만 우선 상세 구현.

## 11. 1차 구현 범위 상세 (Phase 0 + Phase 1)

### Phase 0 — 기반
- `config_store.py`: `config.json` 로드/세이브, 기본값, 위젯별 visible·geometry·설정 스키마.
- `base_widget.py`: 프레임리스·반투명(불투명도 설정)·드래그 이동·모서리 리사이즈·멀티모니터 인지·우클릭 메뉴(닫기/설정/잠금)·geometry 자동 저장.
- `responsive.py`: 위젯 크기 → 폰트/여백 스케일 계산, 내용 구간(breakpoint) 훅.
- `launcher.py`: 시스템 트레이 아이콘, 위젯 on/off 토글, 전역 설정(테마·불투명도·레이아웃 잠금), 종료.
- `main.py`: 런처 기동 + 저장된 위젯 복원.
- PyInstaller 빌드 스크립트 + 시작프로그램 `.bat`.

### Phase 1 — MVP 위젯
- **시계:** 현재 시각 + 날짜 + 요일. 12/24시간·표시 요소 토글. 반응형 폰트.
- **타이머:** 분/초 설정, 시작·일시정지·리셋, 종료 시 알림(소리/시각). 반응형.
- **메모:** 자유 텍스트, 여러 개 인스턴스, 자동 저장. 반응형.

### 검증 기준 (Phase 0+1 완료 조건)
- 3개 위젯을 띄워 자유 이동·리사이즈 시 폰트/레이아웃이 반응형으로 재계산된다.
- 다중 모니터에서 각 위젯을 원하는 모니터에 배치하고 재시작 후 위치가 복원된다.
- 트레이에서 각 위젯을 숨기고 다시 띄울 수 있고, 설정이 보존된다.
- PyInstaller로 빌드한 `.exe`를 파이썬 미설치 환경에서 실행할 수 있다.

## 12. 향후 검토(미결, 후속 Phase에서 확정)

- 출결 월별 Excel 출력 양식(나이스 출석부 그리드 호환 정도) 확정.
- 주간학습계획 API 응답 스키마 확정.
- 자동 업데이트 메커니즘 도입 여부.
