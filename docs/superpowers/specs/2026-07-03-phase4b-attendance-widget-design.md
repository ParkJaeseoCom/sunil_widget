# Phase 4-B — 출결 위젯 설계 문서

- 작성일: 2026-07-03
- 상위 설계: [2026-06-29-teacher-desktop-widgets-design.md](2026-06-29-teacher-desktop-widgets-design.md) §6(위젯 5: 출결)
- 전제: Phase 4-A 완료(RemoteWidget — 단, 본 위젯은 **로컬형이므로 BaseWidget 직접 상속**). 상담기록은 Phase 4-C로 분리.

## 1. 목표·민감정보 원칙

학급 출결을 나이스 기호 체계로 기록하는 **완전 로컬** 위젯. 학생 **이름 없음(번호만)**, 데이터는 PC 로컬 파일에만 저장, **외부 전송 절대 금지**. 나이스 입력 전 임시 기록/증빙 보조가 목적.

## 2. 데이터 모델

- **나이스 16기호** = 상태 4(결석/지각/조퇴/결과) × 사유 4(질병/미인정/기타/출석인정):

| | 질병 | 미인정 | 기타 | 출석인정 |
|---|---|---|---|---|
| 결석 | ♡ | ♥ | ▲ | △ |
| 지각 | # | X | ≠ | ◁ |
| 조퇴 | @ | ◎ | ∽ | ▷ |
| 결과 | ☆ | ◇ | = | ▽ |

- **체험학습 = 결석/출석인정(△)**.
- 저장 파일: `teacher-widgets-data/attendance.json` (config.json과 분리). 구조:
```json
{ "records": { "2026-07-03": { "7": {"status": "결석", "reason": "질병"} } } }
```
  (날짜 ISO 키 → 번호(문자열 키) → {status, reason}. 빈 기록은 키 삭제.)
- 번호 체계: 기존 공유 `class_roster`(남 1~N, 여 51~50+M) 재사용 — `roster_numbers`.
- **저장 안전:** 쓰기 전 기존 파일을 `.bak`으로 복사(1세대 백업). 손상 로드 시 `.bak` 복구 시도.

## 3. 위젯 UI

- **월별 그리드:** 행=번호(class_roster), 열=해당 월 1~말일(주말 열 회색 음영), 셀=기호. 상단 `◀ 2026년 7월 ▶` 월 이동.
- **셀 클릭 → 퀵 메뉴:** `결석 ♡` / `체험학습 △`(최빈 2종 상단) / `지각 ▸`·`조퇴 ▸`·`결과 ▸`·`결석 상세 ▸`(사유 4종 서브메뉴) / `지우기`.
- **자연어 입력줄**(그리드 하단): 예 `7번 6월 29일 결석`, `5번 체험학습`, `12번 7/1 미인정지각`, `7번 결석 취소`. 날짜 생략 시 **"오늘(7/3)로 기록할까요?" 확인 팝업** 후 기록. 파싱 실패 시 입력줄 아래 붉은 안내.
- **Excel 출력**(우클릭 메뉴): ① 현재 월 ② 전체 월. `QFileDialog`로 저장 위치 선택.
- 반응형 폰트(공통), BaseWidget 공통 기능(이동·리사이즈·숨김) 자동.

## 4. 자연어 명령 문법 (결정적 파서 — AI 아님)

```
<번호>번 [<날짜>] <출결어> | <번호>번 [<날짜>] (취소|지움|지우기)
날짜: "N월 M일" | "N/M" | "N.M"  (연도는 현재 표시 월의 연도 기준)
출결어 → (status, reason):
  결석→ (결석, 질병)        체험학습|인정결석 → (결석, 출석인정)
  미인정결석 → (결석, 미인정)  기타결석 → (결석, 기타)
  지각 → (지각, 질병)        조퇴 → (조퇴, 질병)      결과 → (결과, 질병)
  접두 수식(질병|미인정|기타|인정) + (결석|지각|조퇴|결과) 허용 — 예: 미인정지각, 인정조퇴
```
- 파서는 순수 함수 `parse_command(text, today) -> {"number", "date"|None, "status","reason"} | {"number","date"|None,"clear":True}` + 실패 시 `None`.
- 번호가 roster에 없으면 경고(기록은 허용하지 않음).

## 5. Excel 출력 (openpyxl — 신규 의존성)

- **openpyxl** 채택(순수 파이썬, PyInstaller 호환). requirements 갱신 + 빌드 spec hiddenimports 불필요(정적 import).
- 시트 구성(월별): 1행 제목 `YYYY년 M월 출결`, 2행 기호 범례(16종 한 줄), 4행 헤더 `번호 | 1 | 2 | … | 말일`, 이후 행=번호별 기호. 주말 열 회색 채움.
- ① 현재 월 = 시트 1개, ② 전체 월 = 기록 있는 월마다 시트.
- 이름 없음 — 번호만 (개인정보 최소화).

## 6. 컴포넌트 구조

```
core/data_local.py          # (신규) 로컬 민감 데이터 공용: load_json(.bak 복구)/save_json(.bak 백업)
widgets/attendance.py       # 기호표·store 헬퍼·parse_command 순수부 + 퀵메뉴 + Excel export + AttendanceWidget
main.py                     # attendance 등록
```
- Excel 생성 함수는 순수하게: `build_month_sheet(wb, ym, records, numbers)` — 파일 I/O는 위젯의 export 핸들러에서.

## 7. 테스트 전략

- 순수: 기호 조회(16종), store 헬퍼(set/get/clear/월 조회, JSON 라운드트립, .bak 복구), parse_command(정상 각형·수식접두·취소·실패·roster 검증), Excel(워크북을 메모리 생성해 셀 값 검증 — 파일 저장은 tmp_path).
- 위젯(qtbot): roster 기반 그리드 생성, 셀 기호 표시, 월 이동, 자연어 입력 적용(팝업 우회 경로), 데이터 파일 반영.
- 네트워크 없음(로컬 위젯).

## 8. 엣지·에러

- 월 이동 시 해당 월 일수(28~31) 정확 반영(윤년 포함). 미래/과거 월 자유 이동.
- roster 축소 후 기존 기록: 그리드엔 현재 roster 번호만, 데이터는 보존(체크 위젯과 동일 원칙).
- attendance.json 손상: .bak 복구 → 그것도 실패 시 빈 데이터로 시작하되 손상 파일을 `attendance.corrupt-<ts>.json`으로 보존(데이터 소실 방지).
- Excel 저장 실패(권한 등): 상태 라벨 안내, 예외 전파 금지.

## 9. 범위 밖

- 수업일수/마감 계산(학사일정 필요 — 후속), 나이스 자동 입력 연동, 이름 표시(의도적 제외), 상담기록(4-C).
